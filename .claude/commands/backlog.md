# Backlog Command

Read plan.md and find the next unmarked test. Create a backlog item for it.

## Format

```
## [Feature Name]

### Next Test to Implement
- [ ] Test name: [descriptive test name]
- [ ] Expected behavior: [what the test should verify]
- [ ] Acceptance criteria: [specific conditions for passing]

### Implementation Notes
- Minimal code needed: [brief description]
- Dependencies: [any required setup or mocks]
```

## Instructions

1. Read plan.md to find the next unmarked test
2. Create a backlog item following the format above
3. Keep it focused on ONE test only
4. Ensure the test name follows the pattern: should[Action][Condition]
